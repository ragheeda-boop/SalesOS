# Release Notes — v0.2.0

> **Release Date: 2026-07-12 | Status: 🟢 Released**

---

## Overview

v0.2.0 is the **Data Fabric** release, introducing a unified data processing layer that connects Entity Resolution, Hybrid Search, Feature Store, and Knowledge Graph into a single pipeline. Includes significant production hardening across security, performance, and reliability.

**Version codename:** Data Fabric

---

## New Features

### Entity Resolution Pipeline
- pg_trgm fuzzy matching for company name deduplication
- Commercial Registration (CR) number exact-match verification
- Arabic text normalization (Alef variants, diacritics, Ta Marbuta)
- Automated merge: preserve oldest record, enrich from newest
- Confidence scoring (0–1) with configurable thresholds
- Batch candidate detection with pagination

### Hybrid Search (Full-Text + Semantic)
- PostgreSQL tsvector full-text search with GIN index
- pgvector semantic search with HNSW embeddings
- Reciprocal Rank Fusion (RRF) scoring algorithm
- Autocomplete suggestions with prefix and fuzzy matching
- Faceted search with aggregate counts
- Arabic-aware search with normalization pipeline
- Configurable pagination with max_page_size guard

### Feature Store
- Centralized feature repository for entities (company, contact, deal)
- 5 feature types: numeric, categorical, text, embedding, datetime
- TTL-based expiry for time-sensitive features
- Source tracking for feature provenance
- Bulk store/update operations

### Knowledge Graph
- Neo4j-backed graph storage with SQL fallback
- 6 relationship types: OWNS, SUPPLIER, CUSTOMER, COMPETITOR, PARTNER, INVESTOR
- Multi-depth traversal queries (1–3 levels)
- Directional and type-filtered relationship queries
- Weight-based relationship scoring

### Data Fabric Pipeline
- End-to-end orchestration: Ingest → Normalize → Resolve → Enrich → Store → Index
- Pipeline status tracking with per-stage progress
- Partial pipeline execution (select specific stages)
- Pipeline history and duration metrics

## Production Hardening

### Security
- 57 RBAC argument-reversal fixes across all endpoints
- CSRF protection middleware added to all state-changing endpoints
- Auth enforced on previously unprotected demo and metrics endpoints
- Secrets cleanup: hardcoded credentials removed (S5/S8-S10)
- Security audit script: `scripts/security-audit.ps1`

### Performance
- Search timeout guard prevents long-running queries
- tsvector index on company name field for faster full-text search
- Search p95 latency improved: 450ms → 350ms
- Neo4j connection leak fixed (BUG-003)

### Data Integrity
- 7 InMemory repositories migrated to PostgreSQL (TD-001 resolved)
- Search domain PostgreSQL repository (VIO-103)
- Entity Resolution pipeline with merge conflict resolution

### Testing
- Test coverage: 74% → 93% (target: 85%)
- 2054 total tests, 100% pass rate
- New tests: Entity Resolution, Hybrid Search, Feature Store, Knowledge Graph, Data Fabric

## API Changes

### New Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/entity-resolution/match` | POST | Fuzzy entity matching |
| `/api/v1/entity-resolution/merge` | POST | Merge duplicate entities |
| `/api/v1/entity-resolution/candidates` | GET | Get merge candidates |
| `/api/v1/search` | POST | Hybrid search |
| `/api/v1/search/suggest` | GET | Autocomplete suggestions |
| `/api/v1/search/facets` | POST | Faceted search |
| `/api/v1/features/{entity_type}/{entity_id}` | GET | Get entity features |
| `/api/v1/features/{entity_type}` | POST | Store/update features |
| `/api/v1/features/{entity_type}/{entity_id}/{feature_name}` | DELETE | Remove a feature |

| `/api/v1/graph/relationships/{entity_type}/{entity_id}` | GET | Get relationships |

| `/api/v1/data-fabric/pipeline` | POST | Trigger Data Fabric pipeline |
| `/api/v1/data-fabric/status/{pipeline_id}` | GET | Check pipeline status |
