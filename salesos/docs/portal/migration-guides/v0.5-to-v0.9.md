# Migration Guide: v0.5 to v0.9

> **دليل الترحيل من v0.5 إلى v0.9 — ترقية عبر 4 إصدارات رئيسية**

This guide covers migrating from SalesOS v0.5 (Company Intelligence) through all versions to v0.9 (AI & Automation).

---

## Version Path

```
v0.5 ───► v0.6 ───► v0.7 ───► v0.8 ───► v0.9
  │         │         │         │         │
Wave 1    Wave 2    Wave 2    Wave 2    Wave 3
Company   NBA +     Pipeline  Polish    RAG +
Intel.    Opps      Meetings  Release   Workflows
```

---

## v0.5 → v0.6: NBA & Opportunities

### Database

```bash
# Apply new migrations
make migrate

# New tables:
# - opportunities
# - opportunity_features
# - playbooks
# - nba_recommendations
# - nba_feedback
```

### Environment

```bash
# Add to .env
OPENAI_API_KEY=sk-...           # Optional: AI reasoning in NBA
```

### API Changes

| Change | Details |
|--------|---------|
| New base path | `/api/v1/revenue` for opportunities and NBA |
| New permissions | `opportunity:*`, `nba:*`, `pipeline:read` |
| Company pipeline data | Moved from `/companies/{id}/pipeline` to `/revenue/opportunities` |

### Code

```typescript
// OLD — Wave 1
const pipeline = await api.get(`/companies/${id}/pipeline`)

// NEW — Wave 2
const opportunities = await api.get(`/revenue/opportunities?company_id=${id}`)
```

## v0.8 → v0.9: AI & Automation

### Infrastructure

```bash
# Wave 3 requires new services
docker compose -f docker-compose.wave3.yml up -d

# New services:
# - kafka (3 brokers)
# - schema-registry
# - rag-service (ingestion, embedding, retrieval, generation)
# - workflow-engine
# - warehouse (analytics PostgreSQL)
# - etl-pipeline
```

### Database

```bash
make migrate

# New tables:
# - documents, chunks, document_vectors (pgvector)
# - workflows, workflow_executions, workflow_step_executions
# - dim_*, fact_* (warehouse schema)
# - event_store
```

### Environment

```bash
# Required for RAG
OPENAI_API_KEY=sk-...

# Required for Kafka (if using SASL)
KAFKA_BOOTSTRAP_SERVERS=kafka1:9092,kafka2:9092,kafka3:9092
KAFKA_SECURITY_PROTOCOL=SASL_SSL
KAFKA_SASL_USERNAME=...
KAFKA_SASL_PASSWORD=...
```

### API Changes

| Change | Details |
|--------|---------|
| New endpoints | `/rag/*`, `/workflows/*`, `/analytics/*` |
| New permissions | `workflow:*`, `analytics:*`, `rag:*` |
| Feature flags | `ff_workflow_automation`, `ff_rag_pipeline`, `ff_kafka_event_bus` |

### Event System Migration

The in-memory Event Runtime is replaced by Kafka (TD-002):

```
Phase 1: Dual-run (Event Runtime + Kafka in parallel)
Phase 2: Kafka primary, Event Runtime mirror
Phase 3: Event Runtime decommissioned
```

During Phase 1 (2 weeks), both systems run simultaneously. Verify no event loss before decommissioning.

## Rollback Plan

| Version | Rollback Action | RTO |
|---------|----------------|-----|
| v0.9 → v0.8 | `docker compose -f docker-compose.wave3.yml down` + revert DB | 15 min |
| v0.8 → v0.7 | Deploy previous Docker image | 5 min |
| v0.7 → v0.6 | `make migrate-down` to revert tables | 10 min |
| v0.6 → v0.5 | `make migrate-down` + remove revenue permissions | 15 min |

| Item | Check |
|------|-------|

| [ ] Authentication works | `GET /api/v1/auth/me` |
| [ ] Company search works | `GET /api/v1/search?q=test` |
| [ ] NBA works (v0.6+) | `GET /api/v1/revenue/opportunities/{id}/nba` |
