# MUHIDE Sales OS – Notion Automation Suite

## 📋 المتطلبات / Requirements

- Python 3.8+
- حساب Notion مع تكامل (Integration) مفتوح
- صلاحية الوصول إلى قواعد البيانات المطلوبة

## 🔧 التثبيت / Installation

```bash
# 1. تثبيت الحزم المطلوبة
pip install -r requirements.txt

# 2. تعيين توكن Notion
#    - على Windows:
set NOTION_TOKEN=ntn_xxxxxxxxxxxxxxxxxxxx
#    - أو عدّل ملف config.py مباشرة
```

## ⚙️ الإعداد / Configuration

1. افتح ملف `config.py`
2. ضع `NOTION_TOKEN` الخاص بتكامل Notion الخاص بك
3. تأكد من أن معرفات قواعد البيانات (Database IDs) صحيحة
4. يمكنك تعديل `SHEET_ID` لربط Google Sheets (اختياري)

## 🚀 التشغيل / Usage

```bash
python main.py
```

ستظهر قائمة تفاعلية تسمح باختيار البرنامج النصي المراد تشغيله.

## 📂 محتويات المجموعة / Scripts Overview

| الملف | الوظيفة |
|-------|---------|
| `main.py` | القائمة الرئيسية لتشغيل البرامج النصية |
| `config.py` | إعدادات التوكن ومعرفات قواعد البيانات |
| `notion_api.py` | غلاف لمكتبة Notion API (عمليات CRUD) |
| `dedup_scanner.py` | مسح البيانات المكررة عبر المصادر |
| `completeness_scorer.py` | حساب درجة اكتمال البيانات (0-100) |
| `priority_assigner.py` | تعيين الأولويات تلقائياً حسب القواعد |
| `stale_detector.py` | كشف السجلات القديمة (أكثر من 30 يوم) |
| `sfda_sync_checker.py` | التحقق من تراخيص SFDA المنتهية |

## 🗄️ قواعد البيانات / Databases

| المفتاح | قاعدة البيانات |
|---------|---------------|
| `sfda` | الهيئة العامة للغذاء والدواء |
| `engineering_offices` | المكاتب الهندسية |
| `contractors` | المقاولون |
| `suppliers` | الموردون |
| `companies` | الشركات |
| `contacts` | جهات الاتصال |

## 🧠 قواعد تحديد الأولويات / Priority Rules

### SFDA
- الدرجة ≥ 70 + جاهز للتواصل → P1
- الدرجة ≥ 40 → P2
- دون ذلك → P3

### المكاتب الهندسية (EO)
- بيانات كاملة + إشارة إيجابية → High
- بيانات كاملة → Medium
- غير ذلك → Low

### المقاولون
- تصنيف أولى/ثانية + بيانات كاملة → P1
- تصنيف ثالثة/رابعة أو بيانات كاملة → P2
- غير ذلك → P3

### الموردون
- نفس قواعد SFDA
